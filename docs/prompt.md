# 概述
我需要对照FinalShell的功能和界面风格，实现一个类似的项目。名为EShell，使用Tauri2 + React + Tailwind CSS + JS来开发。

# 需求描述

## 1. 服务器SSH连接管理
* **SSH连接配置管理**: 支持添加、删除、修改服务器SSH连接配置，包括主机名、端口、用户名、密码等，要支持配置持久化存储，可存储到json文件。
* **SSH连接列表展示**: 展示所有已配置的SSH连接，包括主机名、用户名、端口等。
* **SSH连接快速登录**: 点击连接列表中的项，快速登录到对应的服务器。

## 2. SSH连接建立和操作
* **SSH连接建立**: 点击连接列表中的项，建立到对应的服务器的SSH连接。
* **SSH连接操作**: 连接成功后，在UI中提供类Term界面执行SSH命令。
* **多Shell管理**： 注意支持通过多个标签页提供多个Shell会话管理，切换标签页更新相关状态的显示，一个服务器可以开多个Shell，不要相互干扰。
* **SFTP连接**: 支持通过SFTP协议上传下载文件。

## 3. SFTP功能支持
* **SFTP上传下载**: 支持通过SFTP协议上传下载文件。
* **SFTP目录浏览**: 支持在UI中浏览服务器上的目录结构，参考FinalShell的布局，一侧为目录树，一侧为当前目录的内容，在每次点击的时候请求目录内容，目录树只需要提供右键菜单中主动刷新的功能。
* **文件实时编辑与回传**: 支持在UI中编辑文件内容，实时将编辑内容回传给服务器。

## 4. UI风格
* **模式切换**: 预置配色方案可供切换，比如白天黑夜模式，如果可以，可尝试加入term壁纸的功能
* **尺寸调节**: 要求大部分边框可以拖动调节尺寸，比如SSH连接操作区域的高度、SFTP目录浏览区域的宽度等，参考VScode的实现。

## 5. 服务器状态显示
* **总状态显示**： 显示服务器的当前内存/总内存、当前CPU占比、各个网卡的网络上下行（支持不同网卡切换，默认显示第一个网卡）
* **进程显示**： 实时显示前四个高占用进程，展示占用内存、CPU、进程ID、命令等信息。
* **磁盘显示**： 显示服务器的各级目录硬盘占用情况，一侧显示路径，另一侧显示已用大小/总大小
* **实时刷新**： 以上所有状态都要实时刷新，默认每5秒刷新一次

## 6. 避免阻塞
* **异步操作**: 由于前端是单线程的，所有耗时操作（如SSH连接、SFTP操作）都要异步执行，避免阻塞UI线程。
* **进度显示**: 对于耗时操作，要在UI中显示进度条或提示，告知用户操作正在进行中。

## 7. 脚本管理
* **脚本列表展示**: 展示所有已添加的脚本，包括脚本名称、路径等。
* **脚本添加**: 支持添加新的脚本，包括脚本名称、路径等。
* **脚本删除**: 支持删除已添加的脚本。
* **脚本执行**: 点击脚本列表中的项，执行对应的脚本。

## 8. AI助手
* **自定义配置**: 支持用户自己选择兼容openai配置的API，包括API Key、Base URL等，要支持配置持久化存储，可存储到json文件。
* **问答功能**： 支持用户在UI中输入问题，调用openai的API获取回答，显示在UI中。
* **读写功能**: 支持用户通过AI直接把命令写入终端，也支持AI读取终端结果。

## 9. 性能优化
* **缓存**：可以把服务器状态的最新信息缓存在前端，然后在切换到其他Shell会话时把这个信息保存下来，下次切换回来的时候先读出缓存内容，再请求新内容
