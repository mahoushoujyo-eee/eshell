## 第一章 系统简介

### 1. 背景

在日常运维工作中，工程人员通常需要同时使用多个工具完成远程登录、命令执行、文件传输、状态监控和问题分析。传统方式存在以下痛点：

1. 终端、文件工具、监控工具分散，切换成本高。  
2. 多服务器并行处理时，容易出现会话混淆和上下文丢失。  
3. AI能力逐步用于运维问答，但缺少“高风险命令人工确认”机制。  

为提升运维效率并降低误操作风险，开发了eShell桌面运维工作台软件。

### 2. 简介

eShell是一款基于 **Tauri 2 + React + Rust** 的桌面应用软件，当前版本号为 **0.1.0**。软件围绕“统一运维工作台”目标，集成以下能力：

1. SSH连接配置管理与多会话操作。  
2. 基于PTY的交互式终端。  
3. SFTP远程文件浏览、上传、下载与编辑。  
4. 服务器运行状态采集与可视化展示。  
5. 脚本管理和一键执行。  
6. AI运维助手（Ops Agent）及命令审批闭环。  

软件后端通过Rust实现核心能力，前端基于React进行交互编排，使用Tauri命令调用机制实现桌面端前后端联动。

### 3. 目标用户

本软件主要面向以下用户群体：

1. Linux服务器运维工程师。  
2. SRE/DevOps工程师。  
3. 需要远程维护环境的后端开发与测试人员。  
4. 希望在本地桌面端集中处理运维任务的技术团队。  

### 4. 主要功能

系统提供终端运维所需的核心功能模块：

1. SSH配置管理和多会话并行连接。  
2. 交互式PTY终端输入输出。  
3. SFTP文件管理、在线查看与保存。  
4. CPU、内存、网卡、进程、磁盘状态监控。  
5. 脚本配置及会话内执行。  
6. AI运维助手问答、只读工具自动执行、写操作审批执行。  
7. 本地JSON数据持久化和会话记录管理。  

### 5. 优势与创新

#### 5.1 会话隔离与多终端并发机制

系统为每个SSH会话维护独立的当前目录、终端输出缓存和状态缓存，通过会话级状态管理避免多标签并行时出现上下文串扰，并在会话异常中断后提供自动重连能力，以保障长时运维场景下的连续性和稳定性。  

#### 5.2 “可读自动、可写审批”的AI执行闭环

AI助手将工具动作分为`read_shell`与`write_shell`两类：只读诊断命令可自动执行并回传结果，涉及写操作的命令则进入待审批队列，由用户确认后执行，且执行结果会回写到会话上下文，从而形成可追踪、可审计的操作闭环。  

#### 5.3 轻量本地持久化与兼容迁移机制

系统采用`.eshell-data`目录进行本地结构化存储，将配置、脚本、AI配置、会话摘要与会话详情分文件管理，并内置历史文件迁移机制，以保证版本升级过程中数据能够平滑延续。  

#### 5.4 桌面端集成化运维能力

系统在单一界面中整合终端、文件、监控、脚本和AI协同能力，相较于多工具分散式操作方式，可有效减少界面切换与上下文重建成本，提升整体运维效率。

## 第二章 系统运行环境

### 1. 系统硬件环境

#### 1.1 客户端最低配置

1. CPU：双核2.0GHz及以上。  
2. 内存：4GB及以上。  
3. 磁盘：可用空间500MB及以上。  

#### 1.2 客户端推荐配置

1. CPU：四核及以上。  
2. 内存：8GB及以上。  
3. 磁盘：可用空间1GB及以上。  

#### 1.3 被运维主机要求

1. 支持SSH服务并具备网络可达性。  
2. 建议为Linux系统，具备常用运维命令环境（如`top`、`ps`、`df`等）。  

### 2. 系统软件环境

#### 2.1 客户端运行环境

1. 操作系统：Windows 10/11、macOS、主流Linux发行版。  
2. 软件形态：Tauri桌面应用。  

#### 2.2 开发与构建环境

1. Node.js 18及以上。  
2. Rust stable工具链。  
3. Tauri 2相关构建依赖。  

#### 2.3 网络环境

1. 能访问目标SSH服务器。  
2. 使用AI助手时，需访问OpenAI兼容接口服务。

## 第三章 系统主要功能（用户）

### 1. SSH配置管理与多会话连接

系统提供SSH配置的新增、编辑、删除和列表展示功能。用户可保存主机地址、端口、用户名、密码及备注信息，并基于配置快速建立会话。系统支持多会话并行，便于同时管理多台服务器。

### 2. PTY交互终端功能

系统基于SSH通道创建PTY会话，提供接近原生Shell的交互体验。用户可实时输入命令并接收增量输出，终端窗口尺寸变化会同步到远端PTY，确保命令行界面显示正确。

### 3. SFTP文件管理与在线编辑功能

系统支持远程目录浏览、文件上传、文件下载、文本文件读取与写回。用户可在界面中直接打开远程文本文件进行编辑，编辑结果自动保存到远程目标路径，提升配置修改效率。

### 4. 服务器状态监控功能

系统可采集并展示服务器运行状态，包括CPU占用率、内存使用情况、网络接口流量、Top进程及磁盘占用信息。状态面板支持周期性刷新，并结合缓存机制减少切换会话时的等待时间。

### 5. 脚本中心与快捷执行功能

系统支持维护脚本定义（脚本名称、脚本路径或直接命令），并可在当前会话中一键执行。该功能适用于重复性巡检、部署检查、日志收集等标准化运维场景。

### 6. AI运维助手与命令审批功能

系统内置Ops Agent，支持多轮会话、会话切换和流式回答。AI助手可按策略规划工具调用：

1. `read_shell`：自动执行只读诊断命令并返回分析结果。  
2. `write_shell`：生成待审批动作，用户确认后执行，拒绝则终止。  

该机制在提升问题定位效率的同时，确保高风险操作具备人工审核环节。

### 7. 数据持久化与会话管理功能

系统在本地目录中持久化关键数据，包括SSH配置、脚本定义、AI配置、会话摘要、待审批动作和消息详情。会话标题支持自动生成，历史数据可在重启后恢复使用，满足日常追溯与连续操作需求。
