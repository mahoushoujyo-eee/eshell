# eShell 1.4.0

Release date: 2026-05-09

## Highlights

- Added an Ops Agent runtime gateway that chooses between `direct_reply`, `lite`, and `pro` execution.
- Simple greetings, API smoke tests, and basic conversational requests can now answer directly without planner/ReAct/multi-agent overhead.
- Auto mode now makes one routing decision across direct reply, lite ReAct, and pro multi-agent workflow.
- Pro mode now runs planner, executor, reviewer, validator, and final answer without an extra duplicate gateway step.
- Reworked conversation compaction to be non-destructive:
  - visible chat history remains unchanged
  - private model summaries are stored under `.eshell-data/ops_agent_context_summaries/`
  - repeated compaction rolls the previous summary forward with newer raw messages after `sourceMessageId`
  - the summary prompt avoids nested summary-of-summary drift
- Runtime compaction now runs before gateway routing, so direct replies, lite runs, and pro workflows share the same model-context policy.
- Added Chinese project README and refreshed root documentation.

## Upgrade Notes

- Existing visible Ops Agent conversation files are preserved.
- New private model-context summaries are written to `.eshell-data/ops_agent_context_summaries/` when long conversations exceed the configured context window.
- Existing `.eshell-data/ops_agent_conversations/` and `.eshell-data/ops_agent_attachments/` content remains compatible.
