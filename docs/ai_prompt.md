# 概述
需要实现一个简单的Agent，有着基本的多轮对话、工具调用，以及相关的多次对话的管理切换等功能，目的在于协助用户做运维相关的shell任务。

# 需求描述
## 依照OpenAI标准接口实现相关功能
## 实现基本的AI对话功能
1. 流式传输：前后端基于流式，实时显示和渲染内容
2. 多轮对话支持：支持进行多轮对话
3. 多次对话的上下文管理：保存每段历史对话，持久化到文件里，同时保证对话可重新加载并继续聊天
4. 工具调用支持：进行读shell和写shell的工具调用，让AI可读写shell。
	 - 读shell的定义：读取“当前屏幕中的终端上下文”，不仅是最后一条输出。
	 - 读shell必须包含：
		 1) 用户已输入的命令内容（按时间顺序）
		 2) 每条命令的执行反馈（stdout/stderr、成功或失败状态、退出码）
		 3) 当前会话的操作上下文（最近一段命令与输出的连续记录，保证AI能理解前因后果）
		 4) 多标签场景下，明确当前活动Shell会话，并仅读取该会话上下文（避免串会话）
	 - 写shell的安全要求：每次写入shell前，必须在会话界面弹出“确认执行/拒绝执行”提示，用户确认后才允许执行。
